# Цель

Реализовать синхронное воспроизведение аудио между пользователями комнаты.

Требования:

* один пользователь является host
* остальные клиенты получают realtime команды
* аудио НЕ передается между клиентами
* каждый клиент самостоятельно загружает аудио из S3/CDN
* воспроизведение должно быть максимально синхронным между устройствами
* использовать scheduled playback
* использовать общий server clock
* использовать drift correction
* минимизировать latency и jitter

# Текущая проблема

Сейчас проект использует:

* HTMLAudioElement
* локальное время клиента
* play/pause события без общего clock
* instant playback
* периодические seek synchronization

Из-за этого:

* drift между устройствами
* рассинхрон через 1–5 минут
* проблемы на mobile/bluetooth
* задержки из-за network jitter
* нет deterministic playback

# Clock synchronization

Реализовать mini-NTP synchronization.

Клиент должен:

* вычислять RTT
* вычислять server offset
* использовать performance.now()
* хранить synchronized server time

Формула:

serverNow = performance.now() + serverOffset

# Scheduled playback

Вместо:
"play now"

Использовать:
"play at exact future server timestamp"

Пример:

{
type: "PLAY",
trackId: "track-1",
position: 42.5,
startAtServerTime: 1715523000500
}

Клиент вычисляет delay:

delay = startAtServerTime - serverNow

и запускает:

source.start(audioContext.currentTime + delay, position)

# Audio loading

Все клиенты:

* загружают одинаковый файл из S3/CDN
* preload before playback
* decodeAudioData before start
* поддерживают buffer cache

Нужно:

* вынести audio loading layer отдельно
* реализовать track preloading
* реализовать decoded buffer cache
* минимизировать decode lag

# Drift correction

Реализовать soft drift correction.

Нельзя постоянно использовать seek.

Использовать:
source.playbackRate.value

Пример:

* если клиент отстает → 1.01
* если клиент спешит → 0.99

После correction:

* вернуть 1.0

Hard resync только если:
delta > 300ms

# Room synchronization

Host:

* управляет playback
* send play/pause/seek/changeTrack

Clients:

* никогда не являются source of truth
* только apply server state
